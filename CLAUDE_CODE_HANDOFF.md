# Claude Code handoff — move the Bloqr Design System to its own repo

This package **is** the Bloqr AI design system. The task is to publish it as the
standalone repository `Bloqr-Systems/bloqr-design-system` and wire the existing
products to consume it. Everything here is repo-ready.

## What this is
- Single source of truth for the Bloqr visual language: `styles.css` + `tokens/`,
  self-hosted fonts in `fonts/`, React primitives in `components/`, brand assets in
  `assets/`, specimen cards in `guidelines/`, and full-screen recreations in `ui_kits/`.
- `package.json` is configured to publish as `@bloqr/design-system` (GPL-3.0).
- `readme.md` → see the **Distribution & versioning** section for the full rationale.

## Do this
1. **Create the repo** `Bloqr-Systems/bloqr-design-system` (private or public per org policy).
2. **Push these contents** to it as the initial commit. The `_ds_*` files and `export/`
   are git-ignored (build/preview artifacts) — regenerate the bundle in CI if consumers
   fetch it directly.
3. **Tag `v1.0.0`** and publish the package:
   ```sh
   npm publish        # @bloqr/design-system, public access
   ```
   (A barrel `components/index.js` is generated by the design-system compiler — if it is
   not present in this snapshot, add one that re-exports every `components/**/*.jsx`
   named export, or wire your bundler's entry to do the same. `package.json#exports`
   already points `.` at `./components/index.js`.)
4. **Wire the consumers** (replace each vendored `brand/` copy with the dependency):
   - **`Bloqr-Systems/bloqr-landing`** (Astro/Svelte): `npm i @bloqr/design-system`,
     link `@bloqr/design-system/styles.css`, import primitives where used. Delete the
     local `brand/` design-system copy; keep the prose brand docs or move them here.
   - **`Bloqr-Systems/bloqr-compiler` → `/frontend`** (Angular 21 + Material): import
     `@bloqr/design-system/styles.css` and the tokens; reimplement the primitives as
     Angular components against the **same CSS custom properties** (the React
     `components/` are the visual contract, not a runtime dep for Angular). Icons: Lucide.
   - **`Bloqr-Systems/bloqr-compiler` → `/docs`**: link `styles.css`, use tokens +
     `guidelines/` specimens.
5. **Cross-reference back:** add a one-line "Design system: `@bloqr/design-system`
   (`Bloqr-Systems/bloqr-design-system`)" note to each consumer repo's README.

## Fidelity note
The `ui_kits/` and `templates/` HTML are **high-fidelity design references** — recreate
them in each product's existing framework using its patterns, not by shipping the HTML.
Tokens, fonts, and `assets/` are the real shippable artifacts.

## Fonts caveat
Space Grotesk is the real self-hosted variable font. Inter + JetBrains Mono currently
load from Google Fonts for preview — **self-host them before production** to honor
Bloqr's no-external-DNS posture.
